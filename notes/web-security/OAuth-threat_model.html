        Minimum OAuth / OIDC security checklist
        
        
        
          
        Redirect URIs: exact registered callback match after safe normalization; no wildcard, substring, or open redirect chain.
        
          
        State, nonce, and flow binding: use high-entropy state for response correlation and UI restoration; use OIDC nonce for ID-token replay protection; bind values to the initiating browser session as defense-in-depth.
        
          
        PKCE: require S256 PKCE for public clients and use it for modern authorization-code flows; PKCE proves possession of the original code verifier.
        
          
        Callback page: no third-party content; no tokens in URL fragments; remove code from URL after exchange.
        
          
        Referrer policy: use no-referrer or strict-origin on OAuth-sensitive pages.
        
          
        Clickjacking: apply CSP frame-ancestors and require explicit user consent for authorization actions.
        
          
        OAuth access tokens: validate issuer, audience, expiry, scopes, and token integrity by JWT validation or introspection before API access.
        
          
        OIDC login / account linking: validate ID-token issuer, audience, azp when present, signature, expiry, nonce, and subject; bind local accounts to issuer plus subject, not email alone unless email_verified is true and the provider is trusted.
        
          
        Scopes: enforce required scope and resource ownership server-side per endpoint.
        
          
        Token lifecycle: single-use codes, refresh-token rotation, replay detection, and transactional locking.
        
          
        Endpoint hardening: SSRF allowlists, no debug output, safe Host handling, and internal-network blocking.
        
          
        Client/mobile: validate TLS, protect deep links, and do not log or expose codes, tokens, client secrets, or refresh tokens.
